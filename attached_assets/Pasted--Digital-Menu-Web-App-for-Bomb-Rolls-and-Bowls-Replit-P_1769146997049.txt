## Technical Requirements

### Mobile Optimization (Primary Focus)
- Responsive design: Mobile-first (320px - 480px primary)
- Touch-friendly buttons (min 44px tap targets)
- Smooth scrolling and transitions
- Fast loading times
- Optimized images (lazy loading)
- No horizontal scrolling

### Technology Stack
- **Frontend**: HTML5, CSS3, Vanilla JavaScript (or React if preferred)
- **Styling**: CSS Grid/Flexbox for layouts, CSS animations
- **Images**: 
  - Use Unsplash API for food images: `https://source.unsplash.com/400x300/?indian-food,rolls` (modify keywords)
  - Or use Lorem Picsum with overlay filters
  - Or DiceBear/placeholder services with food emojis
- **Icons**: Use emoji or Font Awesome for veg (🟢)/non-veg (🔴) indicators

### Features to Implement
1. Smooth page transitions (fade/slide effects)
2. Category accordion expansion/collapse
3. Auto-rotating image carousel with manual controls
4. Veg/Non-Veg filter toggle
5. Sticky header on scroll
6. "Back to Top" button
7. Loading animations
8. Hover effects on all interactive elements
9. Price display with rupee symbol formatting

### Design Specifications
- **Item Cards**: Yellow background, white borders, rounded corners, subtle shadows
- **Buttons**: Yellow with red text, or red with yellow text, 3D effect with shadows
- **Spacing**: Generous padding (16-24px) for touch-friendly UI
- **Font Sizes**: 
  - Headers: 24-32px
  - Item names: 16-18px
  - Descriptions: 14px
  - Prices: 16px bold
- **Borders**: Checkered pattern (10px squares, alternating red/white)

### User Experience
- One-tap category expansion
- Clear visual hierarchy
- Easy price scanning (right-aligned prices)
- Visual separation between veg/non-veg items
- Description text in collapsed state, full details on expand
- Smooth, delightful animations (not overwhelming)

## Deliverables
- Fully functional, responsive web app
- Clean, commented code
- Optimized for mobile devices (iOS/Android browsers)
- Fast load times (<2 seconds)
- Works offline after initial load (optional PWA features)
- No external dependencies if possible (vanilla JS preferred)
